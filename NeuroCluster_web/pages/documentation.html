Overview

NeuroCluster Streamer (NCS) is a high-performance streaming clustering algorithm designed for real-time data processing. Unlike traditional clustering methods that require complete datasets, NCS processes points one at a time with sub-millisecond latency while maintaining exceptional clustering quality.

⚡

Ultra-Fast Processing

Process over 6,300 data points per second with P95 latency under 0.22ms, enabling real-time analytics and instant decision making.

🧠

Adaptive Intelligence

Dynamic threshold adaptation and concept drift detection allow continuous learning and adaptation to changing data patterns.

🎯

Superior Quality

Achieve clustering quality scores of 0.918+ with multi-layer outlier detection and intelligent cluster management.

📊 Performance Metrics: NCS has been tested on datasets ranging from 1K to 10M points, maintaining consistent performance and quality across all scales.

Python Example

Python - Basic Clustering 
from neurocluster import NCSClient

# Initialize the client
client = NCSClient(
    api_key="your-api-key-here",
    base_url="https://api.neurocluster.com"
)

# Prepare your data points
points = [
    [1.0, 2.0, 3.0],
    [1.1, 2.1, 3.1],
    [5.0, 6.0, 7.0],
    [5.1, 6.1, 7.1]
]

# Process the points
result = client.process_points(points)

# Analyze results
print(f"Found {len(result.clusters)} clusters")
print(f"Detected {len(result.outliers)} outliers")
print(f"Processing time: {result.processing_time_ms:.2f}ms")
print(f"Algorithm quality: {result.algorithm_quality:.3f}")

# Examine each cluster
for cluster in result.clusters:
    print(f"\nCluster {cluster.id}:")
    print(f"  Center: {cluster.center}")
    print(f"  Size: {cluster.size} points")
    print(f"  Quality: {cluster.quality:.3f}")

JavaScript Example

JavaScript - Async/Await Usage 
import { NCSClient } from 'neurocluster-streamer';

async function clusterData() {
    // Initialize client
    const client = new NCSClient({
        apiKey: 'your-api-key-here',
        baseUrl: 'https://api.neurocluster.com'
    });

    // Prepare data
    const points = [
        [1.0, 2.0, 3.0],
        [1.1, 2.1, 3.1], 
        [5.0, 6.0, 7.0],
        [5.1, 6.1, 7.1]
    ];

    try {
        // Process points
        const result = await client.processPoints(points);
        
        console.log(`✅ Found ${result.clusters.length} clusters`);
        console.log(`🎯 Processing time: ${result.processingTimeMs}ms`);
        console.log(`📈 Algorithm quality: ${result.algorithmQuality}`);

        // Display cluster details
        result.clusters.forEach((cluster, i) => {
            console.log(`\n📊 Cluster ${cluster.id}:`);
            console.log(`   Center: [${cluster.center.join(', ')}]`);
            console.log(`   Size: ${cluster.size} points`);
        });

        if (result.outliers.length > 0) {
            console.log(`\n🔴 Found ${result.outliers.length} outliers`);
        }

    } catch (error) {
        console.error('❌ Clustering failed:', error.message);
    }
}

clusterData();

API Reference

Base URL

https://api.neurocluster.com

Process Data Points

POST /v1/process-points
Process a batch of data points through the NCS clustering algorithm. Returns cluster assignments, outlier detection, and performance metrics.

Request Parameters

Parameter Type Required Description
points array[][] Required Array of data points, where each point is an array of numerical values
parameters object Optional Algorithm configuration parameters (see Algorithm Parameters section)
return_details boolean Optional Include detailed cluster information in response (default: false)

Algorithm Parameters

NeuroCluster Streamer provides several configurable parameters to optimize performance for your specific use case and data characteristics.

Parameter Default Range Description
base_threshold 0.71 0.3 - 0.9 Base similarity threshold for cluster assignment. Lower values create more clusters.
learning_rate 0.06 0.01 - 0.2 Rate of adaptation for cluster centroids. Higher values adapt faster to changes.
outlier_threshold 0.20 0.1 - 0.5 Threshold for outlier detection. Lower values detect more outliers.
max_clusters 30 5 - 100 Maximum number of clusters to maintain. Prevents memory growth.
decay_rate 0.002 0.001 - 0.01 Rate of cluster confidence decay over time. Helps with concept drift.

Parameter Tuning Guidelines

🎯

High Precision Clustering

base_threshold: 0.8-0.9
outlier_threshold: 0.15-0.25
Best for applications requiring very tight, precise clusters.

🌊

Streaming/Dynamic Data

learning_rate: 0.08-0.15
decay_rate: 0.005-0.01
Optimized for rapidly changing data patterns.

⚡

High-Performance Mode

max_clusters: 10-20
base_threshold: 0.65-0.75
Maximizes throughput for real-time applications.

Performance Optimization

Batch Size Optimization

For optimal performance, consider these batch size recommendations based on your use case:

  • Real-time processing: 1-10 points per request for sub-millisecond latency
  • Streaming applications: 10-100 points per request for balanced throughput
  • Batch processing: 100-1000 points per request for maximum throughput
  • High-volume ingestion: 1000+ points per request (may increase latency)

Memory Management

💾 Memory Usage: NCS uses bounded memory architecture with typical usage of 12-50MB depending on the number of active clusters and data dimensionality.

Monitoring & Metrics

Key performance metrics to monitor in production:

Performance Monitoring Example 
# Monitor key performance metrics
status = client.get_algorithm_status()

print(f"Throughput: {status.throughput_points_per_second}")
print(f"P95 Latency: {status.p95_latency_ms}ms") 
print(f"Memory Usage: {status.memory_usage_mb}MB")
print(f"Clustering Quality: {status.clustering_quality}")
print(f"Active Clusters: {status.active_clusters}")

# Set up alerts for performance degradation
if status.p95_latency_ms > 1.0:
    print("⚠️ High latency detected!")
if status.clustering_quality < 0.8:
    print("⚠️ Low clustering quality!")

Deployment Guide

Production Best Practices

🔒

Security

  • Store API keys securely using environment variables
  • Use HTTPS for all API communications
  • Implement proper authentication and authorization
  • Regular API key rotation
📊

Monitoring

  • Set up performance metric dashboards
  • Configure alerts for latency and error rates
  • Monitor clustering quality trends
  • Track API usage and rate limits
⚡

Scaling

  • Implement connection pooling for high throughput
  • Use async/await patterns for non-blocking operations
  • Consider load balancing for multiple instances
  • Implement proper error handling and retries

Environment Configuration

Environment Variables 
# Production environment variables
export NCS_API_KEY="your-production-api-key"
export NCS_BASE_URL="https://api.neurocluster.com"
export NCS_TIMEOUT="5000"
export NCS_MAX_RETRIES="3"
export NCS_BATCH_SIZE="100"

export NCS_OUTLIER_THRESHOLD="0.20"
🚨 Important: Never commit API keys to version control. Always use environment variables or secure key management services in production.
